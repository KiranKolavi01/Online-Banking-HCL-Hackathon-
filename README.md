# Online Banking System — HCL Hackathon 🏦

A modern, highly secure, and multi-layered RESTful API banking system built for the HCL Hackathon. This platform features strict Role-Based Access Control (RBAC), robust business logic validation, and atomic database transactions to guarantee financial integrity.

---

## 🚀 The Tech Stack

- **Backend:** Python, FastAPI, Pydantic, Uvicorn
- **Frontend:** Streamlit, Pandas (for data tables)
- **Database:** SQLite (Relational, SQL-based)
- **Security:** Bcrypt (Password Hashing), Session tracking

---

## 🔐 Key Features by Role

The system is strictly partitioned into three isolated roles. Users cannot cross-access API endpoints belonging to another role.

### 👥 Customer (Public users)
- **Secure Registration:** Registers with an encrypted password.
- **Account Dashboard:** View issued checking/savings accounts and their net balances.
- **Fund Transfers:** Transfer money between accounts securely. Transfers are guarded by multi-step validation to prevent overdrafts or hacking attempts.
- **Transaction History:** View all processed deposits and deductions.
- **Support Ticketing:** Raise a service request if there is an issue (e.g., lost card or frozen account).

### 🛡️ Admin (Root level bank staff)
*Note: Admins cannot be created via the public portal. They must be provisioned internally.*
- **Customer Issue:** Onboard and KYC customers directly.
- **Account Issuance:** Issue "Savings" or "Current" accounts to customers with a starting positive balance.
- **Global Ledger:** View all transactions processed by every customer in the entire bank in real-time.

### 🎧 Support (Customer Service)
- **Ticketing Hub:** View all service requests generated by customers.
- **Resolution Control:** Update ticket statuses strictly through a defined lifecycle (`pending` ➔ `in-progress` ➔ `resolved`).

## 🧠 Architecture & Business Logic Highlights

This project avoids generic approaches by implementing hardcore business and financial guarantees:

1. **Atomic Transactions (ACID):** During a fund transfer, the deduction from the sender and the deposit to the receiver are locked in a single SQL Transaction (`BEGIN` & `COMMIT`). If the server crashes mid-transfer, it triggers a `ROLLBACK`. No money is ever lost in the void.
2. **Pydantic Validation:** The API strictly catches bad mathematical inputs (like transferring negative numbers) at the Schema level, returning `HTTP 422` before the code even runs.
3. **Overdraft Protection:** It is impossible to transfer more money than what is inside an active account.
4. **State Persistence Front-end:** Fixed dynamic Streamlit refresh issues by binding session state heavily to URL `query_params`, delivering a flawless Single Page Application (SPA) experience.

## 🛠️ How to Run Locally

### 1. Start the Backend API
Open your terminal and navigate to the `backend` folder. Setup the database and launch the FastAPI server.
```bash
cd backend
python database.py        # Initializes the SQLite database tables
uvicorn main:app --reload # Starts the server on port 8000
```
> **API Docs:** You can explore the interactive API Swagger UI at [http://127.0.0.1:8000/docs](http://127.0.0.1:8000/docs)

### 2. Start the Frontend Dashboard
Open a **new** terminal tab, move into the `frontend` directory, and launch the Streamlit App.
```bash
cd frontend
streamlit run app.py      # Starts the dashboard on port 8501
```
> **UI Dashboard:** Access the Bank UI at [http://localhost:8501](http://localhost:8501)
